PESDO Project - JavaScript File Structure Guide
================================================

All JavaScript files in this project are located in the 'src/' directory.
This is a React application using JSX (JavaScript XML) format, so files have
.jsx extension instead of .js (though both work).


ROOT LEVEL JAVASCRIPT FILES
============================

src/main.jsx
------------
- Entry point of the application
- Renders the App component into the DOM
- Location: src/main.jsx

src/App.jsx
-----------
- Main application component
- Sets up routing (React Router)
- Wraps entire app with AuthProvider
- Location: src/App.jsx

src/supabase.js
---------------
- Supabase client configuration
- Database connection setup
- Location: src/supabase.js

src/firebase.js
--------------
- Firebase configuration (if used)
- Location: src/firebase.js


AUTHENTICATION & CONTEXT
=========================

src/contexts/AuthContext.jsx
----------------------------
- Authentication logic
- User profile management
- Login, signup, logout functions
- User session management
- Location: src/contexts/AuthContext.jsx

=============================

FILE EXTENSIONS EXPLANATION
============================

.jsx vs .js:
------------
- .jsx = JavaScript XML (React components)
- .js = Plain JavaScript
- Both work, but .jsx is preferred for React components
- This project uses .jsx for components and .js for utilities

IMPORTANT NOTES
================

1. All JavaScript code is in the 'src/' directory
2. React components use .jsx extension
3. Utility functions can use .js extension
4. Main entry point is src/main.jsx
5. Routing is configured in src/App.jsx
6. All pages are React components in src/pages/
7. Reusable logic is in src/services/ and src/utils/
8. Custom hooks are in src/hooks/
9. Shared components are in src/components/


HOW TO FIND A SPECIFIC FEATURE
================================

Looking for authentication code?
→ src/contexts/AuthContext.jsx

Looking for employer job submission?
→ src/pages/Employer/EmployerDashboard.jsx (handleSubmitJob function)

Looking for jobseeker application?
→ src/pages/Jobseeker/JobseekerDashboard.jsx (handleApplyToJob function)

Looking for admin job approval?
→ src/pages/Admin/JobManagementSimplified.jsx (handleApproveJob function)

Looking for email notifications?
→ src/services/emailService.js

Looking for SMS notifications?
→ src/services/smsService.js

Looking for real-time updates?
→ src/hooks/useRealtimeData.js

Looking for notification UI?
→ src/components/NotificationButton.jsx

Looking for activity logging?
→ src/utils/activityLogger.js


SUMMARY
=======

All JavaScript files are organized in the 'src/' directory with clear
separation of concerns:

- Pages: User-facing components (src/pages/)
- Services: Business logic (src/services/)
- Hooks: Reusable React hooks (src/hooks/)
- Components: Reusable UI components (src/components/)
- Utils: Helper functions (src/utils/)
- Contexts: React context providers (src/contexts/)

This structure makes it easy to find and maintain code.
